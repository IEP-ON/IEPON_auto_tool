🔥 CRITICAL FIX: 확장프로그램 팝업 포커스 문제 해결 - 자동입력 전 팝업을 물리적으로 닫아야 함

[핵심 문제]
- 확장프로그램 팝업이 최상단에 있으면 대상 웹페이지가 비활성화 상태로 유지됨
- DOM 조작과 이벤트는 발생하지만, 웹페이지가 포커스를 받지 못해 실제 UI에 반영되지 않음
- 사용자가 수동으로 웹페이지를 클릭하면 갑자기 작동하는 이유가 바로 이것

[해결 방법]
1. chrome.tabs.sendMessage()로 메시지 전송 시 응답을 기다리지 않음 (await 제거)
2. 메시지 전송 직후 500ms 대기 후 window.close()로 팝업 즉시 닫기
3. Content Script에서 팝업 닫힘 대기 시간 추가 (500ms)
4. 이렇게 하면 대상 웹페이지가 최상단으로 올라와 포커스를 받음

[추가 개선사항]
- setValue() 함수를 마우스 클릭 + 키보드 입력 시뮬레이션으로 변경
- 사람이 입력하는 것처럼 한 글자씩 입력하며 이벤트 발생
- 모든 타이밍 딜레이 최적화 (행 추가, 월 선택, 필드 입력 등)

[확장프로그램 개발 시 필수 주의사항]
⚠️ Chrome Extension Popup은 항상 최상단 레이어에 위치함
⚠️ Popup이 열려있는 상태에서는 대상 페이지가 실질적으로 비활성화됨
⚠️ DOM 조작은 가능하지만 실제 사용자 인터랙션이 필요한 입력은 작동하지 않음
⚠️ 자동화 작업 시작 후 즉시 팝업을 닫아야 대상 페이지가 활성화됨
⚠️ sendMessage() 응답을 기다리면 팝업이 닫히지 않으므로 비동기 처리 필요

이 문제는 모든 Chrome Extension 자동화 프로젝트에서 발생할 수 있는 공통적인 이슈임.
