✅ FIX: 평가계획 필드 UI 커밋 문제 해결 및 타이밍 최적화

[문제]
- 평가계획(마지막 필드) 입력 후 대시보드 UI에서 값이 사라지는 현상 발생
- 필드 입력은 되지만 NICE 시스템의 UI 렌더링이 완료되지 않은 상태에서 다음 작업 진행

[원인]
- 마지막 필드 입력 후 포커스가 해제되지 않아 값이 커밋되지 않음
- NICE 시스템의 change/blur 이벤트 핸들러가 값을 저장하는 시간 부족

[해결책]
1. 평가계획 입력 후 명시적 blur() 호출 (200ms 대기)
2. UI 커밋 대기 시간 추가 (800ms)
3. document.body.click()으로 모든 필드에서 포커스 완전 해제 (300ms 대기)
4. 저장 전 최종 대기 시간 증가 (1000ms)

[타이밍 최적화]
- 평가계획 입력 후 총 2.3초 여유 시간 확보
- NICE 시스템의 비동기 렌더링 완료 보장
- 저장 버튼 클릭 시 모든 필드 값이 UI에 반영된 상태 보장

[테스트 결과]
✓ 평가계획 필드 값이 대시보드 UI에 정상 표시됨
✓ 모든 필드 입력 후 저장 시 데이터 손실 없음
✓ 다중 행 입력 시에도 각 행의 평가계획이 정상 저장됨
